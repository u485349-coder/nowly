Nowly Documentation

How Nowly turns free time overlap into real hangouts.

Nowly is not a calendar tool and not an event planner. It is a live social coordination system built to help friends discover shared openings, act on them quickly, keep private conversation warm, and move between phone and browser without losing momentum.

What the product is for

Spot when your people are actually free, share recurring windows, send a low-pressure prompt, and turn a moment into a real link-up.

What the product avoids

Heavy scheduling, complex event setup, planning theater, and productivity-app energy.

Core stack

Expo app for phone and browser, Express API, PostgreSQL, Prisma, Socket.io, Twilio-ready OTP, push notifications, Railway hosting.

Quick start

The ideal Nowly flow is fast. The product should feel like a live social radar, not a multi-step planner.

  1. Join with your phone number, add your name and city, then pull in your real crew through contacts or invites.
  2. Set a lightweight availability signal like Free now, Free later, or Down this weekend, or add recurring weekly/monthly windows in Now Mode.
  3. Add optional context such as vibe, budget, intent, custom mode names, and live duration so matches feel more human without adding friction.
  4. Wait for overlap notifications or open the app to see who is free nearby right now.
  5. Send a low-pressure prompt like quick bite, coffee run, pull up for a bit, or write your own custom prompt.
  6. Use private chats or group chats to keep the line warm between hangs, and use a hangout thread only when a specific plan is moving.
  7. Share a booking link if you want someone to pick from your recurring availability, then confirm the hang happened so the app gets smarter over time.

Core product features

Onboarding and graph building
  • Phone-first signup keeps entry simple and mobile-native.
  • Contacts and deep-link invites help create friend density early.
  • Optional Discord linking helps reduce cold-start friction.
Availability signals
  • Users set states like Free now, Free later, Busy, or Down this weekend.
  • Signals support optional vibe, radius, energy, intent, custom labels, and live duration inputs.
  • Every signal expires automatically so the app stays fresh and real-time.
Recurring availability and booking links
  • Users can save weekly and monthly recurring windows with exact start and end times.
  • Those windows generate best-fit mutual overlap suggestions.
  • Users can share a booking link so someone else can pick from the specific availability they chose.
Overlap matching engine
  • The backend checks friend overlap continuously and ranks the strongest opportunities.
  • Time, proximity, responsiveness, recurring-fit, and lightweight social signals influence match quality.
  • Matches are meant to feel timely, not random.
Low-pressure proposals
  • Prompts are casual by design: quick bite, coffee run, walking nearby, 10 min only, plus custom prompts in the user's own words.
  • Response flows stay lightweight so people do not feel trapped in a full RSVP commitment.
  • This lowers anxiety and makes real hangs more likely to happen.
Private chats and group chats
  • Friends can open persistent 1:1 chats with a chat bubble action.
  • Users can create reusable private group chats with multiple friends.
  • These chats are separate from hangout threads so conversation can stay alive even when no plan is active.
Micro-group coordination
  • Users can ask Who's free tonight? and pull a small group together quickly.
  • Threads support chat, reactions, ETA, and location sharing for real-time motion.
  • The system is optimized for fast social coordination, not long planning chains.
Recap and rhythm
  • After the hangout window, Nowly asks if the hang happened.
  • Completed hangs move into recap/past-hangs mode instead of staying live.
  • That signal powers recaps, responsiveness scores, and rhythm cues so the app gets smarter about who links up, when they link up, and what tends to convert.

How people use it well

Best practices for users
  • Keep signals honest and lightweight instead of trying to plan the whole day.
  • Use casual activity language so invites feel easy to say yes to.
  • Invite the same real-world cluster first: one friend group, campus, or neighborhood.
Best practices for growth
  • Nowly gets stronger as local friend density increases.
  • The highest-leverage loop is invite, overlap, proposal, hangout, recap, repeat.
  • Winning one tight social pocket matters more than broad empty reach.

Backend and deployment snapshot

Production-minded backend
  • Node.js + Express API with Prisma on PostgreSQL.
  • Socket.io powers live thread behavior.
  • Twilio Verify handles phone OTP when configured.
  • Push notifications support overlap alerts, proposals, and group movement.
Hosting model
  • Deploy the backend on Railway with a Postgres service in the same project.
  • Deploy the website on Vercel from the docs folder.
  • Deploy the Expo app as a browser app for web and point both phone and browser to the same live backend URL once the API is healthy.
